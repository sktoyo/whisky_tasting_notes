# Whisky Tasting Note Web – MVP Definition (Local-First)

## 1. MVP 목적
- 개인용 위스키 테이스팅 노트를 **작성 / 저장 / 탐색 / 감상**할 수 있는 MVP 완성
- 기능 수보다 **UI/기능 기획 검증 + 빠른 반복(피드백 루프)**을 우선
- 이후 필요 시 **PostgreSQL + Docker로 이식** 가능한 구조를 염두에 둔다

---

## 2. 구현 전략 (Local-First)
- 1차 MVP는 **로컬 환경에서 단독 실행**을 목표로 한다.
- 저장소는 MVP 단계에서 다음 중 하나를 사용한다.
  - 권장: SQLite (단일 파일 DB)
  - 대안: JSON 파일 저장 (초경량)
- 이미지 업로드는 로컬 폴더(`/uploads`)에 저장한다.
- PostgreSQL / Docker는 MVP 필수 요건이 아니며, Phase 2에서 도입한다.

---

## 3. MVP에서 반드시 되는 것 (In Scope)
- 테이스팅 노트 CRUD (Create / Read / Update / Delete)
- Nose / Palate / Finish를 **키워드 + 상세 설명 구조**로 기록
- 기본 키워드 + 사용자 커스텀 키워드 혼용
- 위스키 기본 정보 기록
- 대표 이미지 업로드 (1장)
- 임시 저장(Draft) 기능
- 테이스팅 노트 게시판
  - 카드 뷰 / 리스트 뷰
- 정렬 기능
- 검색 기능 (AND / OR 조건 조합)
- 오늘의 추천 노트 5건
  - 하루 동안 고정되는 랜덤 선정
  - 전체 노트 수가 5건 미만이면 있는 만큼 표시
- 테이스팅 노트 내용을 텍스트 파일(.txt)로 Export

---

## 4. MVP에서 하지 않는 것 (Out of Scope)
- 로그인 / 회원 관리
- 댓글, 좋아요, 팔로우
- 점수/조회수 기반 추천 알고리즘
- 관리자 UI (승격은 개념만, UI는 제외)
- 모바일 앱 전용 UI
- 배포/운영 리허설 (Docker/Compose 포함)

---

## 5. 핵심 페이지 구성
1. 테이스팅 노트 작성 / 수정 페이지
2. 테이스팅 노트 상세 페이지
3. 테이스팅 노트 게시판 페이지

---

## 6. MVP 성공 기준
- 신규 사용자가 설명 없이 노트 작성 → 저장 → 게시판 탐색 가능
- NPF 키워드 구조가 직관적으로 이해됨
- “오늘의 추천 노트”가 새로고침해도 하루 동안 동일
- 로컬 실행만으로도 전체 기능이 동작
- UI/기능이 확정되면 Phase 2로 Postgres/Docker 이식 가능

---

## 7. Phase 2 (후속)
- PostgreSQL 도입
- Docker / docker-compose로 재현 가능한 실행 환경 구성
- 마이그레이션/백업/배포 리허설
